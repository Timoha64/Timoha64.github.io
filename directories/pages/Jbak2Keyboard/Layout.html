Понятие «Раскладка» В узком смысле раскладка представляет из себя один‐единственный xml файл. Но раскладка может ссылаться на ∙ Иконки клавиш(см. @wikiarticle_2821) ∙ Текстовые файлы(см. @wikiarticle_2817) ∙ Файлы замен(см. @wikiarticle_2807) ∙ Другую раскладку(см. @wikiarticle_2820) Чтобы сослаться на файл, находящийся в папке jbak2Keyboard/keyboards, в качестве значения атрибута, который ссылается на этот файл, укажите полное имя файла(вместе с точкой и форматом после неё). Вы также можете создавать свои собственные папки в папке jbak2Keyboard/keyboards; в них вы можете создавать другие папки, располагать во всех этих папках файлы, в общем, создавать свою собственную иерархию папок и файлов в них. Чтобы сослаться на файл, находящийся в вашей собственной иерархии, в качестве значения атрибута, который ссылается на этот файл, укажите путь к файлу. Например, у вас в папке jbak2Keyboard/keyboards есть папка {55FFFF Тексты}. Тогда, чтобы сослаться на файл {55FFFF Файл1.txt}, находящийся в этой папке {55FFFF Тексты}, укажите значением атрибута {55FFFF Тексты/Файл1.txt}. Допустим же, что у вас в этой папке {55FFFF Тексты} есть папка {55FFFF Информация}, а в этой папке файл {55FFFF Файл2.txt}. Тогда значение атрибута, который ссылается на {55FFFF Файл2.txt}, будет следующим: {55FFFF Тексты/Информация/Файл2.txt}. И т.д. Итак, код раскладки может ссылаться на какой‐либо внешний файл. В таком случае, логично, указанный в коде раскладки внешний файл должен существовать. Если его не будет существовать, то раскладка не будет функционировать в полной мере как это было задумано создателем раскладки. Поэтому, в широком смысле раскладка представляет из себя не только один xml файл, но и все другие файлы, на которые ссылается этот xml файл.
